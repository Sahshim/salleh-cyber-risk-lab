## 1. Who Owns the Output of an AI Agent Using a Proprietary Prompt Framework?

### The Core Problem

When an AI agent generates output — code, a risk report, a governance diagram — using a *proprietary prompt framework* (like the Zero-Trust AI Agent Governance Framework developed in this lab), ownership of the resulting output is legally ambiguous under current Singapore law.

### Singapore Copyright Act 2021 — Key Provisions

Under the **Copyright Act 2021 (Singapore)**, copyright subsists in original works that are:
- Expressed in a material form
- The result of *human authorship*

The Act does not explicitly address AI-generated works. There is **no sui generis right** for computer-generated output in Singapore — unlike the UK (CDPA 1988, s.9(3)), which grants copyright to the "person who undertakes the necessary arrangements."

### Analysis — Three Scenarios

**Scenario A: Human uses a proprietary prompt to generate output**  
If a human operator crafts and deploys a structured prompt (e.g., a governance audit template), and the output requires significant human judgment to select, arrange, and interpret — a court *may* find sufficient human authorship in the selection process. The prompt itself may be protected as a **literary work** if it exhibits sufficient originality.

**Scenario B: Autonomous agent generates output without human review**  
Where the agent acts autonomously (tool calls, memory retrieval, output generation with no human in the loop), there is likely **no copyright protection** in Singapore for the output. The output falls into the public domain by default.

**Scenario C: Hybrid — human-designed framework + autonomous execution**  
This is the most commercially significant case. The *framework* (the governance ruleset, permission model, risk classification schema) may be protected as:
- A **literary work** (if documented)
- A **trade secret** (if maintained in confidence — see Section 3 below)
- A **patentable invention** (if it constitutes a technical process — see Section 4)

The *output* generated by the agent remains legally unclaimed under current Singapore law.

### Practical Implication for This Lab
The prompt governance templates and risk classification schemas developed here should be **documented, dated, and kept in version control** as evidence of authorship and prior creation. This establishes a baseline for trade secret and copyright claims if the framework is commercialised.

## 2. Is the Zero-Trust AI Agent Governance Framework Protectable as a Trade Secret?

### Relevant Law: Singapore Trade Secrets — Common Law & PDPA Overlap

Singapore does not have a standalone trade secrets statute. Protection arises under:
- **Breach of confidence** (common law, *Coco v AN Clark* test)
- **Computer Misuse Act** (unauthorised access to confidential systems)
- **Employment contracts** and NDAs (contractual protection)

### The *Coco v AN Clark* Three-Part Test (as applied in Singapore courts)

To succeed in a breach of confidence claim, the information must:
1. **Have the necessary quality of confidence** — it must not be publicly known
2. **Have been imparted in circumstances importing an obligation of confidence**
3. **Have been used without authorisation to the detriment of the owner**

### Can a Governance Framework Qualify?

| Element | Assessment |
|---|---|
| Quality of confidence | ✅ Yes — if not publicly published and documented as proprietary |
| Circumstances of confidence | ✅ Yes — if shared only under NDA or in controlled settings |
| Detriment from misuse | ✅ Yes — commercial value of the framework is identifiable |

**Key risk:** Publishing the framework on GitHub (public repo) **destroys** the quality of confidence. Once public, it cannot be reclaimed as a trade secret.

### Recommended Control Strategy
- Maintain a **private repo** for the actual framework ruleset
- Publish only the *concept and architecture* publicly (this repo)
- Use a **README notice**: "Framework architecture is illustrative. Implementation specifications are confidential."
- Require written NDA before sharing implementation details

## 3. Patent Eligibility of AI Governance Systems — Singapore Position

### Relevant Law: Patents Act (Singapore), s.13

An invention is patentable if it is:
- **Novel** (not part of prior art)
- **Inventive** (non-obvious to a person skilled in the field)
- **Industrially applicable**

### Exclusions Relevant to AI

Under s.13(2), the following are *not* inventions for patent purposes:
- A scheme, rule, or method for performing a mental act
- A program for a computer *as such*

However, if an AI governance framework produces a **technical effect** beyond the normal physical interactions of running software (e.g., it measurably reduces system-level security incidents, enforces hardware-level sandboxing, or alters network-level permission states), it *may* cross the threshold for patentability.

### IPOS Examination Practice

IPOS follows the **European Patent Office (EPO)** approach for software-related inventions. A claim framed as a *method of controlling an AI agent's tool execution* with a defined technical character is more likely to succeed than a claim framed as a *governance policy* or *risk model*.

### Strategic Recommendation
For this lab's framework, the most viable IP pathway is **not patent** (difficult, expensive, slow) but rather:
- **Trade secret** for the implementation specifications
- **Copyright** for the documented framework as a literary work
- **Trademark** for the brand identity (name + logo) if commercialised

## 4. Copyright in Training Data — Risk Surface for AI Agent Operators

### Why This Matters for AI Governance

An organisation deploying AI agents that perform web scraping, document ingestion, or retrieval-augmented generation (RAG) over third-party content faces **secondary copyright liability** if the agent reproduces protected material without licence.

### Singapore Position

The Copyright Act 2021 introduced a **text and data mining (TDM) exception** (s.244) for computational data analysis, but it is **limited to non-commercial research**. Commercial AI agent deployments that ingest and reproduce copyrighted content do not benefit from this exception.

### Risk Taxonomy for AI Agent Operators

| Agent Action | Copyright Risk Level | Control |
|---|---|---|
| Summarising public web content | Medium | Paraphrase, don't reproduce |
| Ingesting licensed documents for RAG | High | Verify licence scope |
| Reproducing verbatim text in output | Critical | Filter layer required |
| Training a model on scraped data | Critical | Legal review required |
| Generating novel output from ingested context | Low–Medium | Monitor for substantial similarity |

### Governance Control: Output Filtering
The Zero-Trust governance framework should include an **output audit layer** that flags verbatim reproduction of ingested source material. This is both a copyright risk control and an AI governance best practice.
